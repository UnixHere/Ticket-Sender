# 🎟 QR Ticket Sender & Verifier

A two-part system for sending tickets by email and checking attendance at an event.

- **Ticket Sender** (`main_svg.py` + `prepare_svg.py`) — reads an Excel file, generates a personalised SVG ticket with a QR code, exports it as a PDF, and sends it by email via [Resend](https://resend.com) including a calendar invite
- **Ticket Verifier** (`webapp.py`) — a locally-hosted web app that scans QR codes at the entrance and marks arrivals directly in the Excel file

## Excel Structure

Both tools share the same `students_database.xlsx` file:

| Column | Header | Description |
|--------|--------|-------------|
| A | Name | Student's full name |
| B | Class | Class (e.g. `4.A`) |
| C | ID | Unique 3-digit number — use `generate_ids.py` if missing |
| D | Email | Student's email address |
| E | Sent | Set to `1` by Ticket Sender after sending |
| F | Arrived | Set to `1` by Ticket Verifier when scanned |

Columns E and F are created automatically on first run if they don't exist.

## .env Configuration

Create a `.env` file in the same folder as the scripts:

```env
RESEND_API_KEY=re_xxxxxxxxxxxxxxxxxxxx
SENDER_EMAIL=tickets@yourdomain.com
SENDER_NAME=Organiser
EVENT_NAME=Graduation 2026
EVENT_DATE=Friday, 30 May 2026
EVENT_TIME=18:00
EVENT_LOCATION=Main Hall, High School
EVENT_DURATION_MINUTES=120
```

**Accepted `EVENT_DATE` formats:** `DD.MM.YYYY`, `YYYY-MM-DD`, `DD/MM/YYYY`, or a long-form date like `Friday, 30 May 2026`.

`EVENT_DURATION_MINUTES` is optional (default: 120). Sets the event duration in the calendar invite.

> **Never commit `.env`** — it contains API keys. Make sure it's in `.gitignore`.

## 📤 Ticket Sender

### 1. Preparing the SVG Template

The `prepare_svg.py` tool adjusts your Figma export so it works with the system.

```bash
python prepare_svg.py your_ticket.svg
```

The script will ask you:
1. Which text element is the **student name** → replaces it with `{NAME_PLACEHOLDER}`
2. Which is the **class** → replaces it with `{CLASS_PLACEHOLDER}`
3. Optional **X offset** for each placeholder (shift right/left in pixels)
4. Whether you want to add a **QR sidebar** on the right (recommended)

The result is saved as `ticket_template.svg`.

> **Important:** The Figma export must contain actual `<text>` elements. If you used "Outline text", the text will have been converted to paths and the script won't find it. Re-export without that option.

If students don't have IDs yet, generate them:
```bash
python generate_ids.py
```

### 2. Preview — Check the Result Without Sending

In `main_svg.py`, keep `MODE = "preview"` (default) and run:

```bash
python main_svg.py
```

A `ticket_preview/` folder is generated with files for each student:
- `ticket_Name.svg` — generated ticket
- `ticket_Name.pdf` — final PDF (this is what gets sent)
- `ticket_Name.ics` — calendar invite

### 3. PDF Text Offset Calibration

Playwright renders the SVG through a real Chromium browser, which ensures faithful reproduction of the Figma design. Occasionally the print renderer shifts text slightly differently. To calibrate:

```bash
python main_svg.py adjustpdf
```

This opens `ticket_preview/_adjust_test.pdf` (first student only). Adjust in `main_svg.py` based on the result:

```python
TEXT_Y_OFFSET = -10  # negative = up, positive = down
```

Repeat until the text sits correctly.

### 4. Real Sending

When you're happy with the result, change in `main_svg.py`:

```python
MODE = "real"
```

And run:

```bash
python main_svg.py
```

- Only students without `1` in column E are included — already-sent students are skipped
- The system asks for confirmation (`YES`) before sending
- After each successful send, column E is updated to `1`
- There is an automatic 1.5 second pause between emails

Each email contains:
- **HTML body** with event information
- **PDF attachment** — personalised ticket with QR code
- **ICS attachment** — calendar invite (Google Calendar, Apple Calendar, Outlook)

### 5. Resending to a Specific Student

```bash
python main_svg.py resend 001
python main_svg.py resend "John Smith"
python main_svg.py resend john@school.com
```

Searches by ID, name, or email (case-insensitive). If `MODE = "real"`, asks for confirmation.

## 📷 Ticket Verifier

A mobile-friendly web app you run on a laptop at the entrance. Any phone on the same WiFi can open it and use it as a QR scanner.

### Features

- 📷 **QR Scanner** — uses the phone camera to scan tickets
- ✅ **Instant verification** — checks the scanned QR against the Excel database
- ⚠️ **Duplicate detection** — warns if a ticket has already been scanned
- ✋ **Check-in button** — marks student as arrived in column F
- ↩ **Undo** — un-check a student if checked in by mistake
- 🔍 **Manual lookup** — search by name, ID, class, or email
- 📋 **Attendees list** — live stats and filters
- 📱 **QR code for the app URL** — prints a scannable QR in the terminal on startup

### Running

```bash
python webapp.py
```

The terminal will show the local IP address and a QR code:

```
=======================================================
  🎟  Ticket Verifier
=======================================================
  ✓  Database: students_database.xlsx

  Open in browser (same WiFi):
  → https://192.168.1.42:5000

  Scan this QR code to open on any device:
  ...
  (Or visit https://192.168.1.42:5000/qr to download the QR as an image)
=======================================================
```

Open the address on a phone connected to the **same WiFi**.

> iPhone: use Safari for camera access  
> Android: use Chrome for camera access

### API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/verify` | Verify a QR string. Body: `{ "qr": "Name \| Class \| ID" }` |
| `POST` | `/api/mark_arrived` | Check in or undo. Body: `{ "row": 5, "arrived": true }` |
| `GET` | `/api/stats` | Returns `{ total, arrived, pending }` |
| `GET` | `/api/attendees` | Returns the full student list |
| `GET` | `/qr` | PNG QR code pointing to the app URL |

## QR Format

Both tools use the same QR code format:

```
Name | Class | ID
```

For example: `Jane Smith | 4.A | 142`

The verifier matches on **ID** (exact) and does a soft name check — a mismatch shows a warning but does not reject the ticket.
